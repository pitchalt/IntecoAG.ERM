Описание смысла задачного подхода.

1. Для каждого бизнес-объекта и каждое бизнес-действие с ним создаётся пара управляющих объектов - задача и администратор задачи.

2. Задачи показывается пользоватлею в списке его задач Он может быврать какую-либо из них и выполнить её, либо отменить её и т.п.

3. В результате производится действие над самим бизнес-объектом, скажем, он переходит в другой статус или посылается оповещениео
состоянии бизнес-объекта и т.п.

4. Административная задача предназначена для упралвения присоединённой к ней задачей. Например, в ней можно настроить списки пользователей,
которым доступен просмотр задачи, которым доступно выполнение задачи и т.п. В административной задаче может быть указан приоритет присоединённой задачи, 
крайний срок её решения, ответственный пользователель и т.п.

5. Задача позволяет порождать посредством фабрики (Callback на метод класса бизнес-объекта) произвольное количество бизнес-объектов.
